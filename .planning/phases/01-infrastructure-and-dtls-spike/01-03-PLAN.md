POST /api/hue/pair
  Request: { "bridge_ip": "192.168.1.x" }
  Response 200: { "status": "paired", "bridge_ip": "...", "bridge_name": "..." }
  Response 403: Link button not pressed
  Response 502: Bridge unreachable

GET /api/hue/status
  Response 200: { "paired": true/false, "bridge_ip": "...", "bridge_name": "..." }

GET /api/hue/configs
  Response 200: [{ "id": "...", "name": "...", "status": "...", "channel_count": N }]
  Response 400: Bridge not paired

GET /api/hue/lights
  Response 200: [{ "id": "...", "name": "...", "type": "..." }]
  Response 400: Bridge not paired

GET /api/health
  Response 200: { "status": "ok" }

    2. Create Frontend/src/components/PairingFlow.tsx:
       - State machine with steps: "checking" | "unpaired" | "awaiting_button" | "pairing" | "paired" | "error"
       - On mount: call getBridgeStatus(). If paired, go to "paired" step showing bridge name and entertainment config list.
       - "unpaired" step: Show instructions ("1. Press the link button on your Hue Bridge. 2. Enter the bridge IP address below. 3. Click Pair within 30 seconds."). Input for bridge IP. "Pair" button.
       - "pairing" step: Show spinner/loading text "Pairing..."
       - "paired" step: Show green checkmark + "Paired with {bridge_name} at {bridge_ip}". Call getEntertainmentConfigs() and display as a list with name and channel count. (BRDG-05: entertainment config selection — display for now, selection wiring comes in Phase 3).
       - "error" step: Show error message in red. "Try Again" button returns to "unpaired".
       - On "Pair" click: call pairBridge(). On 403 -> show link button error. On 502 -> show unreachable error. On success -> go to "paired".

    3. Create Frontend/src/components/PairingFlow.test.tsx:
       - Mock global fetch for each test case

       - test_shows_paired_status: mock status as paired with bridge_name, assert "Paired" text and bridge name shown
       - test_shows_error_on_403: mock status unpaired, fill IP input, click Pair, mock pair response as 403, assert error message shown
