       - Config selector dropdown: fetches configs via `fetchConfigs()`, shows entertainment config selection
       - Start/Stop streaming button: reads `isStreaming` from useStatusStore, calls `startStreaming(configId)` / `stopStreaming()` from regions API
       - Light list using shadcn ScrollArea:
         - Each light row shows: name (bold), type (badge), draggable
         - Each row has `draggable` attribute and `onDragStart` setting `e.dataTransfer.setData('lightId', light.id)` and `e.dataTransfer.setData('lightName', light.name)`
         - Visual feedback: cursor grab, slight opacity on drag
       - Currently assigned lights section: shows which regions have which lights assigned (read from useRegionStore)
       - Tailwind styling: `flex flex-col gap-3 p-3 border-l h-full`

    2. Wire the `onDrop` handler in `Frontend/src/components/EditorCanvas.tsx`:
       The onDrop handler MUST live on the wrapper div inside EditorCanvas.tsx where `stageRef` is declared. This is because `stageRef.current.setPointersPositions(e)` must be called before `getPointerPosition()` when the event originates from a DOM drag, and stageRef is local to EditorCanvas.

       - On the existing wrapper div around `<Stage>`, wire the `onDrop` handler:
         - `e.preventDefault()`
         - Read `lightId` and `lightName` from `e.dataTransfer.getData()`
         - Call `stageRef.current!.setPointersPositions(e)` to update Konva pointer
         - Get pointer position via `stageRef.current!.getPointerPosition()`
         - Convert to normalized coords: `[pos.x / width, pos.y / height]` (using the width/height props)
         - Find which region contains this point using `pointInPolygon` against each region's denormalized polygon
         - If hit: call `updateRegion(hit.id, {light_id: lightId})` (API), update store
         - Store the light name on the region for label display

       Do NOT put the onDrop in EditorPage — it must be in EditorCanvas where stageRef is accessible.

    3. Update `Frontend/src/components/RegionPolygon.tsx`:
       - Show the assigned light name as a `<Text>` label inside the polygon (positioned at centroid)
       - If region.light_id is null, no label (or show region.name in dim text)
       - To get the light name: either store it on the region object or maintain a lightMap in the store/component

    4. Update `Frontend/src/components/EditorPage.tsx`:
       - Replace the "Light Panel" placeholder with `<LightPanel />`

    5. Optionally update `Frontend/src/api/hue.ts`:
       - If `LightResponse` on the backend gets `segment_count`, add it to the Light interface
       - If not yet available from backend, default to showing type only (segment_count is a Phase 5 concern per BRDG-04)
