    2. Create `Frontend/src/components/EditorCanvas.tsx`:
       - Props: `width: number, height: number` (pixel dimensions of the canvas)
       - Uses `usePreviewWS(true)` to get live camera feed imgSrc
       - Uses `useImage(imgSrc ?? '')` from `use-image` to get Konva-compatible image
       - Declares `stageRef = useRef<Konva.Stage>(null)` — this ref is used for pointer resolution in the onDrop handler (Plan 04)
       - Three-layer Konva `<Stage>`:
         - Layer 0 (preview): `<Image>` with `listening={false}` — redraws on every WS frame
         - Layer 1 (regions): Maps over `useRegionStore.regions`, renders `<RegionPolygon>` for each
         - Layer 1 also renders drawing-in-progress: `<Line>` with dash style showing currentPoints + `<Circle>` at first point as close target
         - Layer 2 (handles): Reserved for vertex anchors of selected region (rendered inside RegionPolygon)
       - Stage `onClick` handler:
         - In 'polygon' mode: get pointer position, check if near first point (distance < 10px) -> commit polygon, else append point
         - In 'rectangle' mode: track mousedown/mouseup for click-drag rectangle (start pos on mousedown, build 4-corner polygon on mouseup)
         - In 'select' mode: let events propagate to RegionPolygon click handlers
       - Stage `onMouseDown`/`onMouseUp`/`onMouseMove` for rectangle drawing mode:
         - onMouseDown: record start position
         - onMouseMove: if drawing, show preview rectangle via `<Rect>` or `<Line>` with 4 corners
         - onMouseUp: commit as 4-vertex polygon (normalized), clear drawing state
       - On polygon commit (both modes):
         - Normalize coords using `normalize(points, width, height)`
         - Call `createRegion({name: 'Region N', polygon: normalizedPoints})` (API)
         - Add returned region to store via `addRegion`
         - Clear drawing state
       - Delete handler (from toolbar):
         - If selectedId is set, call `deleteRegion(selectedId)` (API)
         - Remove from store, clear selection
       - Keyboard: listen for Delete/Backspace key on Stage to delete selected region
       - Escape key: cancel current drawing, clear drawingPoints
       - Load regions on mount: `fetchRegions().then(setRegions)`
       - Wrap the `<Stage>` in a div with `onDragOver={e => e.preventDefault()}` and `onDrop` handler placeholder (Plan 04 wires this). The wrapper div is where HTML5 DnD events are handled because `stageRef` is declared in this component.

       CRITICAL per research:
       - Set `listening={false}` on preview layer (Layer 0)
       - Rectangle mode stores as 4-vertex polygon in data model (no special Rect shape)

    3. Create `Frontend/src/components/EditorPage.tsx`:
       - Layout using Tailwind: `flex` container
       - Left side (~70%): DrawingToolbar above, EditorCanvas below
       - Right side (~30%): Placeholder div "Light Panel" (Plan 04 fills this)
       - Compute canvas dimensions from container width using a `useRef` + `ResizeObserver`
         - Canvas maintains 16:9 aspect ratio based on container width
         - Canvas height = width * (9/16) or match capture aspect ratio (480/640 = 0.75)
       - Pass `onDelete` handler to DrawingToolbar that calls EditorCanvas's delete logic
