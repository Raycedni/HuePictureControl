1. **Add `_capture_reconnect_loop` method** to `StreamingService` in `Backend/services/streaming_service.py`:
   ```python
   async def _capture_reconnect_loop(self) -> bool:
   ```
   - Set `self._state = "reconnecting"` and push state to broadcaster
   - Exponential backoff: delay starts at 1s, doubles each retry, caps at 30s (same as bridge reconnect)
   - In retry loop (while run_event is set):
     - Call `self._capture.release()`
     - Call `await asyncio.to_thread(self._capture.open)` (MUST be in to_thread since cv2.VideoCapture is blocking — Pitfall 3 from research)
     - On success: log info, set state to "streaming", push state, return True
     - On RuntimeError: log warning with delay, `await asyncio.sleep(delay)`, double delay
   - If run_event cleared: return False

2. **Modify `_frame_loop`** in `StreamingService`:
   - Replace the existing RuntimeError handler (lines 266-272) that clears run_event and returns
   - New behavior: on RuntimeError from `get_frame()`, call `await self._capture_reconnect_loop()`
   - If reconnect returns True: `continue` the frame loop (resume streaming)
   - If reconnect returns False: set state to "error", push error state, return
