1. **Extend `Light` interface** in `Frontend/src/api/hue.ts`:
   ```typescript
   export interface Light {
     id: string
     name: string
     type: string
     is_gradient: boolean
     points_capable: number
   }
   ```

2. **Add `ConfigChannel` interface and `fetchConfigChannels` function** in `Frontend/src/api/hue.ts`:
   ```typescript
   export interface ConfigChannel {
     channel_id: number
     segment_index: number
     light_id: string
     light_name: string
     light_type: string
     is_gradient: boolean
     segment_count: number
   }

   export async function fetchConfigChannels(configId: string): Promise<ConfigChannel[]> {
     const response = await fetch(`/api/hue/config/${configId}/channels`)
     // standard error handling
     return response.json()
   }
   ```

3. **Update `LightPanel.tsx`** to show gradient segments and channel counter:

   a. Add state for config channels:
   ```typescript
   const [channels, setChannels] = useState<ConfigChannel[]>([])
   ```

   b. Fetch channels when `selectedConfigId` changes (add useEffect):
   ```typescript
   useEffect(() => {
     if (!selectedConfigId) return
     fetchConfigChannels(selectedConfigId)
       .then(setChannels)
       .catch(err => console.error('Failed to load channels:', err))
   }, [selectedConfigId])
   ```

   c. Group channels by `light_id` to build the segment display structure. For each unique light_id in channels, collect all its channel entries. A light with 1 channel = non-gradient (render as single draggable item, same as today). A light with N>1 channels = gradient (render as parent header + N indented segment rows).

   d. **Channel counter** (always visible, per user decision): Add to the Lights section header area. Count assigned channels: `regions.filter(r => r.light_id !== null).length`. Display as `{assignedCount} / 20 channels`. Style: `text-xs font-mono`. Color: normal for <20, `text-yellow-500` for ==20, `text-red-500` for >20. Use `cn()` for conditional classes.

   e. **Gradient segment rows**: Each segment row is draggable. `onDragStart` sets:
   - `e.dataTransfer.setData('channelId', String(channel.channel_id))`
   - `e.dataTransfer.setData('channelName', `${channel.light_name} - Seg ${channel.segment_index + 1}`)`
   - `e.dataTransfer.setData('lightId', channel.light_id)` (for backward compatibility)
   - `e.dataTransfer.effectAllowed = 'copy'`

   f. **Non-gradient lights**: Continue to use existing drag behavior (lightId + lightName). Also set channelId if available from channels data (find the single channel for this light).

   g. **Gradient parent header**: Show light name + badge "[gradient]" or segment count badge. Not draggable. Segments indented below with `ml-3 border-l-2 border-primary/30`.

   h. **Segment row display**: "Seg {n}" label + "ch {channel_id}" in muted text. Smaller padding than parent light rows.

   i. Keep the existing "assigned" indicator -- if a region has a light_id matching any of the gradient light's channels, show which segment is assigned where.
