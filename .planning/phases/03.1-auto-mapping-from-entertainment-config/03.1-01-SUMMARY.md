## Decisions Made
- `channel_pos_to_screen` maps Hue z-axis to screen y-axis (z is depth/vertical in Hue 3D space, matches screen Y)
- Region IDs use deterministic pattern `auto:{config_id}:{channel_id}` — no UUID needed, enables idempotency
- `persist_channel_regions` separated from `auto_map_entertainment_config` to allow unit testing persistence without bridge calls
- GET /api/regions/ endpoint uses trailing slash to match FastAPI router prefix behavior
