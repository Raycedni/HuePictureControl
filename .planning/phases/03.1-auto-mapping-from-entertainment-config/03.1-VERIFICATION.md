### Overlay Rendering Note

`handleImageLoad` stores `naturalWidth/naturalHeight` in `imgDimensions` state — this is used purely as a load-success guard (non-null = image loaded). The actual overlay pixel calculations in `getOverlayStyle` correctly use `imgRef.current.clientWidth/clientHeight` (rendered dimensions). This matches the SUMMARY decision: "Overlay positioning uses rendered clientWidth/clientHeight so overlays track the displayed image regardless of CSS scaling."

### Trailing Slash Note

`fetchRegions()` calls `fetch('/api/regions')` (no trailing slash). The FastAPI route is `GET /` under prefix `/api/regions`, making the canonical URL `/api/regions/`. FastAPI's default `redirect_slashes=True` issues a 307 redirect from `/api/regions` to `/api/regions/`. Browser `fetch()` follows GET 307 redirects automatically. This works correctly in practice but could be made explicit with `fetch('/api/regions/')` to avoid the round-trip redirect.
