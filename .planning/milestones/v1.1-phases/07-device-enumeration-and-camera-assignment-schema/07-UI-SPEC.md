## Scope Note

Phase 7 is primarily a backend phase (device enumeration API + database schema). It delivers one and only one new UI surface: a degraded-identity-mode alert rendered when the backend reports that sysfs (`/sys/class/video4linux/`) is inaccessible inside Docker/WSL2. All other UI work (camera dropdown, preview switching, zone status) is deferred to Phases 9 and 10. The design contract below is therefore minimal and anchored to the existing system.

## New UI Element: Sysfs-Unavailable Alert

This is the only new UI surface delivered in Phase 7.

### Trigger
Backend responds to `GET /api/cameras` with a field `identity_mode: "degraded"` (or equivalent) indicating sysfs is inaccessible and device identity is falling back to VIDIOC_QUERYCAP card name.

### Placement
Inline banner inside `EditorPage`, rendered above the canvas area — same slot used by the existing 20-channel warning (`assignedCount > 20`).

### Visual Specification
```
bg-amber-500/10 border border-amber-500/25 text-amber-400 text-xs px-3 py-2
```
- Background: `rgba(245,158,11,0.10)` (matches `bg-amber-500/10`)
- Border: `rgba(245,158,11,0.25)` (matches `border-amber-500/25`)
- Text color: `rgb(251,191,36)` (matches `text-amber-400` in dark mode — `--hue-amber` equivalent)
- Text size: 12px (`text-xs`)
- Padding: 12px vertical 8px horizontal (`px-3 py-2`)
- No icon (keeps parity with existing channel-count banner; lucide icon optional if executor judges layout benefit)

### Dismiss Behavior
Non-dismissable. Alert persists as long as the backend reports degraded identity mode. It is informational, not actionable — no button or close control.

### Stacking
If both the 20-channel warning AND the sysfs alert are simultaneously active, render sysfs alert first (above the channel warning). Use `flex flex-col` wrapper to stack.

## Interaction Contracts (API-to-UI Binding)

These are defined here so Phase 9/10 executors consume a consistent contract.

### `GET /api/cameras` response shape (informational — not rendered in Phase 7)

```typescript
interface CameraDevice {
  device_path: string        // e.g. "/dev/video0"
  stable_id: string          // "VID:PID:serial" or card name fallback
  display_name: string       // human-readable, shown in dropdown (Phase 10)
  connected: boolean         // true if device is currently accessible
  last_seen_at: string | null // ISO 8601 timestamp, null if never seen
}

interface CamerasResponse {
  devices: CameraDevice[]
  identity_mode: "stable" | "degraded"  // triggers sysfs alert when "degraded"
}
```

### `POST /api/cameras/reconnect` request shape (informational — not rendered in Phase 7)

```typescript
interface ReconnectRequest {
  stable_id: string
}

interface ReconnectResponse {
  connected: boolean
  device_path: string | null
  display_name: string
}
```
